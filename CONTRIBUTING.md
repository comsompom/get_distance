# Contributing to TargetLock

Thanks for helping improve TargetLock! Please follow these guidelines.

## Getting Started

1. Fork or copy the project.
2. Create a feature branch.
3. Keep changes focused and well documented.

## Code Style

- Swift 5.0+
- Prefer clear, readable naming.
- Keep UI updates on the main thread.
- Avoid force unwraps unless safe by design.

## What to Include in a PR

- Summary of changes
- Screenshots (if UI changed)
- Steps to test
- Any new documentation updates

## Reporting Issues

Include:
- iOS version and device model
- Steps to reproduce
- Expected vs actual behavior
- Screenshots or logs if possible
